SZORP ANALYSIS PAGE

1. Introduction

Our team works hard on this project to create a very enjoyable game to any players.

The SZORP game is a very complex game, yet it's easy to learn for players, but a challenge for the programmers with almost all components having timed events, and different behaviours.

This document provides an insight to the development process of the game with detailed descriptions about the classes and easy to read details about classes.

2. Overview

In this section we will provide the starting point of our project, the skeleton of all underlying elements of the game.

The following diagram is the first prototype of our views about this project. Later we added the Settings class to store settings. And filled out this diagram with children of the classes and with attributes and operations.

2.1. Full Class Diagram

2.2.1. GameManager Class

The Game Manager class controls every non player controlled entities, the time in the game, crop growths, spawners. It also manages save files, and map generation. This is the class that keeps the game intact, without this nothing could work.

2.2.2. TimedObject class

This class manages so called 'timed' objects (effects, entities, crops), it contains the TimePass method, which controls the time passing in all the derived classes.

2.2.3. Entity class

The entity abstract class describes "living and thinking" objects(NPC-s) and the player itself. The player counts as an entity, with the same type of stats as other "living" creatures.

2.2.3/1. Hostile class

This class means any entity that can attack the player, these type of entities only spawn in the maze.

2.2.3/2. Passive class

This class means any entity that can co-exist with the player, and only interacts with the user when provoked.

2.2.3/3. Player class

This class means the players entity, this entity is controlled by the player and when it dies, the player loses the game or loses all items.

2.2.3/4. Fish class

Fish

2.2.4. Spawner class

The spawner class manages resource and entity spawners, which generates collectible resources, and living creatures.

2.2.4/1. EntitySpawner class

In this object entity instantiation happens at given positions.

2.2.4/2. ResourceSpawner class

In this object collectible resource instantiation happens at given positions.

2.2.5. Tile class

The tile class contains map elements(ground and walls) with saving specific information about the tile like style.

2.2.6. Effect class

Effects can be applied on entities, and these effects give bonuses to the entity they are applied on.

2.2.7. Crop class

The crop class means those farming resources that are placed by the player.

2.2.7/1 Bush class

Bushes can be collected multiple times, collecting from a bush sets back the current stage by 3, and after 3 growth it can be collected again for a pre-determined amount.

2.2.7/2 Plant class

Plants can be collected one or infinite amount of times, if infinitely collectible, then collect is available after a cool-down.

2.2.8. Item Class

The item class mean player collectible resources, consumables, armors and weapons.

2.2.8/1. Armor Class

This type of item when in a specific spot in the inventory give armor bonus to the player.

2.2.8/2. Weapon Class

This type of item when in a specific spot in the inventory give strength bonus to the player.

2.2.8/3. Consumable Class

This type of item when used by the player, replenish food and water to the player.

2.2.8/4. Resource Class

This type of item can be used for crafting or repairing other items.

2.2.9. Settings Class

This class stores settings for the game such as volume, graphics and difficulty.

3. Class Declarations

3.1. Game Manager

In the game manager the maps are declared in a form of tile arrays.

Attributes:

Visibility Type Name Description
private int _survivalSize The size of the survival field, this is the X and Y lengths, so if the value is 10, then the map consists of 100 Tiles.
private int _mazeSize The size of the maze, counting from the survival field how long should the maze spread out.
private Tile[] _survivalMap An array containing the survival fields tiles.
private Tile[] _mazeMap An array containing the mazes tiles.
private int _tickSpeed The number of ticks in a second, this value is set to be 6 by default, changing this number speeds up or slows down the game.
